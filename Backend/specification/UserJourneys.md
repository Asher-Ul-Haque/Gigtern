# User Journeys

## Types of Users
1. Driver
2. Passenger
3. Admin

### Stories 

1. Driver while listing a ride
* **Identity**: signs in with IIIT ID 
* **Offer**: enters route, time, seats and price
* **View Status**: View status on active rides

2. Passenger while finding a ride:
* **Identity**: Passenger signs up/logs in 
* **Passenger**: enters their desired route and time 
* **Request**: Passenger click request seat in chosen ride

3. Driver acceptance and transaction flow 
* **Review**: Driver views their pending request list 
* **Accept**: Driver clicks accept on a request 
* **Payment**: Passenger pays the contribution

4. Ride completion and final settlement
* **completion**: Driver marks the ride as complete 
* **Settlement**: Internal platform action (commission)
* **Rating**: reputation update
